Barista Responsive Web App

Objective

To build a responsive web recipes app that allows users to search, upload and share drink recipes



Purpose

The target audience for my app ‘Barista’ is individuals 18-50 who want to make coffee and tea drinks at home. The app will be available across ios and android devices and will be most effectively used while buying ingredients or making drinks



Project Description

Barista is a responsive web app for 18-50-Year-olds that allows users to add, share, and save their favorite coffee and tea recipes



Role

UI/UX Designer



Skills

Competitive Analysis, User Research, User Testing, Design Thinking, Visual Design, Wireframing, Prototyping, Mockups, User-Centered Design



Tools

Adobe XD, Adobe Photoshop, Usability Hub



MVP ( Minimum Viable Product)

At this point in my project, I had to find a way to optimize my features or practice “lean UX”. I had to think critically alongside my user research about the features that were needed in my app vs the features that were not critical to the functionality of the app.



Hypothesis

Users rather make coffee and tea drinks at home because it saves them time and money



Objective

The objective of this research is to build a responsive web app that allows users to search, upload and share drink recipes



Success Metrics

Recipe shares, number of site visits, number of app downloads, number of accounts created



User Stories and Feature Requirements

When I navigate the app I want to upload a recipe so I can allow others to make my drinks

-Support for recipe input -Multi-app connectivity -Catalog for user-made recipes

When I navigate the app I want to search for recipes so I can make new drinks

-Search feature -Drink catalog -Suggested recipes

When I navigate the app I want to create an account so I can be credited for my uploaded recipes

-Form for account creation -Create a sense of security for the use

When I navigate the app I want to find recipe instructions so I can make my drink step by step

-Scroll down instructions under recipes

When I navigate the app I want to share a recipe so I can show my family and friends

-Social sharing feature

Rapid Prototypes

User Testing

Users testing was the most helpful part of this whole UX process. It really put my designs to the test and allowed me to understand my users frustrations and confusions
Scope: I will be testing “Barista” a responsive web app in which signing up/in, uploading and saving, and sharing recipes will be tested.
Schedule: I will be meeting my participants remotely via google meet in order to conduct my tests
Sessions: I will aim for my sessions to last no longer than 15 minutes per participant(3)
Equipment: I will test my low-fidelity wireframes on an Iphone11 prototype, I will use my laptop in order to test users and a notepad to write down behavioral observations
Metrics: I will use the 1-5 error severity rating from Jakob Nielsen

Jakon Nielsen's Error Severity Rating System

UI Design

The user interface design for the Barista app is centered around a youthful and playful approach. To accentuate that look I chose primary colors which are often related to youthful activities. My target audience is very large so I wanted a design that could easily relate to my younger and older users. During my competitive analysis, I realized that many coffee apps usually use colors such as green, brown, and red. I wanted an app my users could easily remember because of its distinct look
